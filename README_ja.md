# Skill-Manager

**組織全体の AI 資産（プロンプト、ツール、設定）を統合し、公式および社内ナレッジの同期・キュレーション・循環を加速させるための管理フレームワーク。**

## 🌟 ビジョン

`Skill-Manager` は、AI エージェントごとの実装の複雑さを抽象化し、**「ユーザーが好みのエージェントを使いながら、プロジェクトの文脈に最適化された組織知にアクセスし、常に最新の AI 進化を享受できる」**環境を提供します。

## 🎯 なぜ Skill-Manager なのか？ (Jobs to be Done)

Skill-Manager は単なるファイル管理ツールではありません。AI エージェントを活用した開発ワークフローにおける「前進」を自動化します。

1. **「最新エコシステムへの追従 (Stay Updated)」**
   - **Job:** 各社（Google, Anthropic 等）が更新し続ける最新のベストプラクティス（プロンプト、ツール定義、MCP 設定、最新実装パターン）を、自分でコードを書くことなく即座に自分の環境へ注入（Inject）したい。
   - **価値:** 常にエージェントを「最新かつ最強の状態」に保ち、追従のための不毛な調査・更新作業から解放されます。

2. **「プロジェクトへの適合と診断 (Contextual Alignment)」**
   - **Job:** 自分の技術スタックや業務ドメインを AI が分析し、膨大なカタログの中から「今、このプロジェクトに本当に必要な能力」だけを診断（Diagnostic）して自動セットアップしてほしい。
   - **価値:** 膨大なアセットの海から探す手間を省き、プロジェクト開始時から「文脈を理解した即戦力」をエージェントに装備させることができます。

3. **「現場知恵の共有と還元 (Knowledge Circulation)」**
   - **Job:** 現場で生まれた優れたパターン（プロンプト、ツール、ルール）を組織全体の資産として手軽に共有（Share）し、他のチームの成功を加速させたい。
   - **価値:** 「知恵の独占」と「車輪の再発明」を撲滅します。個人の成果が組織全体の「生きた土台」へと還元され、チームを越えて知恵が繋がり続けるエコシステムを構築します。

4. **「マルチエージェントの柔軟な活用 (Agent Agility)」**
   - **Job:** 特定のエージェント（Claude, Gemini 等）の構造に縛られず、タスクの特性（長大なコンテキスト、コード生成の正確性等）に合わせて、最適なエージェントを柔軟に選択・組み合わせて活用したい。
   - **価値:** 特定のエージェントへの「ロックイン」を解消します。エージェント（頭脳）を切り替えても、培ってきたプロンプトやツール（道具）は共通の資産としてそのまま使い続けることができます。

## 🏗️ アーキテクチャ: 並列接続モデル

`Skill-Manager` は、並列なディレクトリ構造を通じて、ローカルプロジェクトとグローバルな資産リポジトリ（ハブ）を接続します。**Asset Consultant** は、ハブとプロジェクトの双方に配備され、アセットやリソースの管理と同期を担います。

![Architecture Overview](docs/architecture-overview.drawio.png)

### 🧠 コア・コンセプト

1. **エコシステム同期 (Ecosystem Synchronization)**: 各社の公式リポジトリ（Google, Anthropic 等）を **submodule** として統合し、`scan_assets.py` で適宜スキャン。リサーチの負担なく、常に「世界標準の最新の賢さ」を手元に届けます。
2. **戦略的マッピング (Contextual Intelligence Mapping)**: **Asset Consultant** が「診断・提案」と「構造定義・ガバナンス」の両輪を担います。膨大な資産の中から、プロジェクトの技術スタックやフェーズに「今、本当に必要なもの」を特定・配備し、エコシステムの健全性を維持します。
3. **検証済みの知恵の循環 (Verified Knowledge Circulation)**: 現場での成功パターンを組織の「型」にマッピングし、Hub へと還元する双方向フロー。個人の発見を組織全体の「生きた土台」へと昇華させ、車輪の再発明を撲滅します。
4. **エージェント非依存の抽象化層 (Agent-Agnostic Abstraction)**: エージェントごとの設定ディレクトリ（`.claude/`, `.agents/` 等）を共通構造にマッピングするシンボリックリンク層。頭脳（エージェント）を切り替えても同じ道具（スキル）が動く「ポータビリティ」を実現します。
5. **並列ブリッジ構造 (Bridge-Link Architecture)**: Hub（中央）と Project（現場）を分離し、リンクで結ぶ構造。プロジェクトを軽量に保ちつつ、必要な知能のみをオンデマンドで注入し、中央の改善を全プロジェクトへ即座に反映させます。

### 📂 ディレクトリ構成 (論理コンセプト)

`Skill-Manager` は共通資産とプロジェクト固有の作業を明確に分離します：
- **グローバルハブ**: スキル（公式、社内、サードパーティ）の中央リポジトリ。
- **ターゲットプロジェクト**: 必要なスキルのみをインポートするローカルワークスペース。

物理的なディレクトリ構造や「ブリッジリンクモデル」の詳細については、**[アーキテクチャ設計と戦略 (docs/architectural-design.md)](docs/architectural-design.md)** を参照してください。

## 🚀 はじめかた (Bootstrap Workflow)

`skill-manager` ディレクトリから、わずか 2 ステップでターゲットプロジェクトをセットアップできます。

1.  **ハブの準備**:
    ```bash
    git clone --recursive {this-repo-url}
    cd skill-manager
    python3 core/tools/scan_assets.py
    ```

2.  **ターゲットプロジェクトの初期化**:
    ```bash
    python3 core/tools/setup_project.py --repo {your-target-project-path} --name "My Project"
    ```
    *これにより、`.env` の構成、ディレクトリ構造の作成、および `Asset Consultant` のデプロイが自動的に行われます。*
