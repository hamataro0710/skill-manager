# 🛡️ Agent Governor (知能ガバナンス)

Agent Governor は、エージェント・エコシステムの **可観測性（Observability）** と **完全性（Integrity）** を維持するための中心的な権限を持ちます。すべての配備された資産が、**「3層のライフサイクル・マトリックス」** 内に正しく配置されていることを保証します。

## 🧭 必須プロトコル

### 1. コンテキストの特定 (ユーザーへの確認)
マップを更新する前に、ライフサイクル・フェーズを適合させるためにプロジェクトの性質を **必ず** 確認してください。
- **プロジェクトの種類**: ソフトウェア開発、戦略コンサルティング、学術研究、その他。
- **ワークフロー・フェーズ**: 標準的なフェーズ（プロジェクト計画 → 開発 → テスト → デプロイ）がプロジェクトに適合するか確認。
- **主要な目標**: 到達しようとしている「理想の状態」は何か？

### 2. 監査 & マッピング (3層の整合性)
アクティブなすべての実体を以下の3つのレイヤーにマッピングし、**`.skills/ACTIVE_ASSETS.md`** を維持します：

- **Layer 1: ワークフロー・フェーズ (横軸)**
    - プロジェクトのライフサイクル（例: 計画 → 開発 → テスト → デプロイ）をマッピング。
- **Layer 2: スペシャリスト・スキル (縦軸 / 特定フェーズ)**
    - 各フェーズに特化したエージェントやツールを割り当て（例: 開発には Swarm、テストには Skill Creator 等）。
- **Layer 3: 共通基盤 & セットアップ (基盤インフラ)**
    - 全フェーズにまたがる基盤インフラ資産を配置（例: Plan Navigator, Doc Co-authoring, Agent Governor, Asset Consultant）。

### 3. ライフサイクル・マトリックスの視覚化
`ACTIVE_ASSETS.md` 内の Mermaid 図を、このマトリックスを反映するように更新します。
- **水平フロー**: フェーズの遷移を視覚化。
- **垂直方向の整合性**: どのスペシャリスト・スキルがどのフェーズを支援しているかを表示。
- **基盤による支持**: ライフサイクル全体を支える共通基盤を視覚化。

## 📂 管理対象ファイル (.skills/ 内)
- **`ACTIVE_ASSETS.md`**: フェーズ、スペシャリスト・スキル、共通基盤の確定マトリックス・マップ。
- **`ASSET_INDEX.md`**: 探索用の資産マスターカタログ。

## 🛠 使用トリガー
- **環境の変化**: 新しいスキルのインポートや、プロジェクトのライフサイクル・フェーズを変更した後に実行。
- **能力の照会**: ユーザーが「知能はどのように構造化されていますか？」や「どのツールがどのフェーズを支援していますか？」と尋ねたときに実行。

## 📝 ベストプラクティス
- **インフラ優先**: `agent-governor` と `asset-consultant` は、環境そのものを管理するため、常に **共通基盤（Foundation）** 資産として認識してください。
- **機能的完全性**: 3層マトリックス内のどこにも配置されていないスキルは「管理外」と見なし、フラグを立てるべきです。
