# 🛡️ Agent Governor (知能ガバナンス)

Agent Governor は、エージェント・エコシステムの**可観測性（Observability）**と**整合性（Integrity）**を維持するための中心的な権限を持ちます。AI が利用可能なツールを明確に把握し、現在導入されている能力の「意図」を文書化して保持することを保証します。

## 🧭 必須プロトコル

### 1. 視覚の同期 (インデックス)
監査の前に、AI の「目」が最新であることを確認せよ。
- **アクション**: `python3 core/tools/scan-assets.py` を実行せよ。
- **洞察**: `.skills/ASSET_INDEX.md` を読み込み、`.skills/ASSET_DIFF.md` に記載された新着アセットをユーザーに報告せよ。

### 2. 配備の監査 (アクティブ・アセット)
`.skills/` ディレクトリを精査し、実際に何が配備されているかを確認せよ。
- **アクション**: `.skills/` 内の全エンティティをリストアップせよ。
- **根拠**: 各アセットの役割と採用理由を **`.skills/ACTIVE_ASSETS.md`** に記録・更新せよ。

### 3. 可視化と記録
オーケストレーター、ツール、専門スキルの関係性を明確にせよ。
- **アクション**: `.skills/ACTIVE_ASSETS.md` 内で Mermaid 図を活用し、能力のフローをマップ化せよ。

## 📂 管理対象ファイル (.skills/ 配下)
- **`ASSET_INDEX.md`**: 潜在的な全アセットのカタログ。
- **`ASSET_DIFF.md`**: アセットライブラリの最近の変更点。
- **`ACTIVE_ASSETS.md`**: 現在配備されている能力の確定マップ。

## 🛠 実行タイミング
- **更新後**: サブモジュールが更新された際、または新しいスキルがインポートされた際に実行せよ。
- **システムチェック**: ユーザーが「今の能力は？」や「どのような構成ですか？」と尋ねた際に実行せよ。

## 📝 ベストプラクティス
- **能動的な報告**: `ASSET_DIFF.md` から新着アセットを常に報告し、ユーザーに新しい価値を提案せよ。
- **文脈的整合性**: すべてのアクティブなアセットに対し、`ACTIVE_ASSETS.md` 内で明確な「採用理由」を定義せよ。
