# 🤝 Asset Consultant (統合版)

Asset Consultant は、プロジェクトの**能力拡張（Capability Expansion）**と**エコシステムの完全性（Ecosystem Integrity）**を維持するための中心的権限を持ちます。戦略的なアセット選定と厳格なガバナンスを組み合わせ、高パフォーマンスなエージェント環境を維持します。

## 🧭 クイックレファレンス

| タスク | 主要なアクション / ガイド |
| :--- | :--- |
| **アセットの探索・選定** | 要件を分析し、`.skills/ASSET_INDEX.md` から最適なものを提案。 |
| **新しいアセットの導入** | `python core/tools/import_skill.py <name>` を実行。 |
| **能力マップの更新** | `.skills/ACTIVE_ASSETS.md` を 3 層マトリックスと同期。 |
| **構造の監査 (Audit)** | 各フェーズに対するスペシャリスト・スキルの配置が妥当か検証。 |

---

## 🛠 必須プロトコル

### 1. 探索 & 選定 (コンサルタント・モード)
能力の「追加」や「探索」に関するリクエストでは、**「アセット直結」**ワークフローに従います：
- **EE (EssenceExplore)**: 必要な「機能セット」（例: GitHubアクセス、記憶保持）を特定。
- **CD (ChallengeDiscover)**: 技術スタック（言語、フレームワーク）をスキャンし、制約を把握。
- **AE (ApproachExplore)**: `ASSET_INDEX.md` から最適なアセットの組み合わせを提案。
- **成果物**: すべてのコンサルテーションは、具体的なインポートコマンドの提示で完結せよ。

### 2. マッピング & 整合性 (ガバナンス・モード)
アクティブなすべての実体を以下の 3 つのレイヤーにマッピングし、**`.skills/ACTIVE_ASSETS.md`** を維持します：
- **Layer 1: ワークフロー・フェーズ (横軸)**: 例: 計画 → 開発 → テスト → デプロイ。
- **Layer 2: スペシャリスト・スキル (縦軸)**: 特定フェーズに特化したエージェント（例: `notion-spec`, `linear`）。
- **Layer 3: 共通基盤 (Base)**: 全フェーズを支える共通資産（例: `plan-navigator`, `asset-consultant`）。

### 3. 可視化 (Visualization)
構造が変化するたびに `ACTIVE_ASSETS.md` 内の Mermaid 図を更新します。`asset-consultant` 自身は常に **共通基盤 (Foundation)** 資産としてマークしてください。

## 📂 管理対象リソース
- **`.skills/ASSET_INDEX.md`**: 探索用の資産マスターカタログ。
- **`.skills/ACTIVE_ASSETS.md`**: プロジェクト知能の確定マトリックス・マップ。
- **`core/tools/import_skill.py`**: アセット導入のための主要実行ツール。

## 📝 ベストプラクティス
- **完全性優先**: 3 層マトリックスのどこにも配置されていないスキルは「管理外」と見なし、分類を促すフラグを立てるべきです。
- **フィルタとしての文脈**: プロジェクトの文脈（言語等）はアセットを絞り込むためだけに使用し、一般的なコーディングアドバイスは行うな。
